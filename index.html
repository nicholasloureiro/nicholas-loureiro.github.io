No início deste projeto, criei um novo repositório no GitHub e o clonei para o meu ambiente de desenvolvimento local. Este passo inicial foi crucial para estabelecer as bases do projeto e iniciar seu versionamento. Em seguida, sem a utilização de JavaScript, foquei na preparação da estrutura básica do projeto, criando diretórios específicos para os templates e para os recursos estáticos, como CSS e imagens, essenciais para a interface do usuário. Para armazenar as informações das postagens, como título, descrição e autor, optei por um arquivo JSON, que facilitou a manipulação e o armazenamento dos dados de maneira estruturada. Utilizando o Flask, desenvolvi um sistema capaz de ler os dados dos arquivos markdown e gerar páginas HTML estáticas para cada postagem, transformando assim o conteúdo em markdown em páginas web visualmente atraentes. Um aspecto importante foi a implementação do Gitflow no projeto, criando branches específicas para desenvolvimento, features e releases, o que ajudou a manter o projeto organizado e facilitou o trabalho colaborativo. Além disso, dediquei um tempo para personalizar o estilo do blog com CSS, buscando um design que fosse simples e elegante, proporcionando uma experiência de usuário agradável. Antes de finalizar, realizei testes locais para verificar se as postagens eram geradas corretamente e se o estilo visual estava conforme o planejado. Esses testes foram essenciais para assegurar a qualidade do produto final. Após concluir o desenvolvimento e os testes, fiz o push do projeto para o GitHub e ativei o GitHub Pages, publicando assim o blog na internet. Por último, como parte do projeto, documentei detalhadamente cada etapa, publicando essa documentação como o primeiro post do blog, servindo como um guia prático para quem deseja seguir um caminho semelhante.
