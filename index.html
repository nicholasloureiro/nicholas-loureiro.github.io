Configuração do Repositório Primeiro, criei um novo repositório no GitHub para este projeto e clonei-o para o meu ambiente de desenvolvimento local. Este passo foi essencial para iniciar a estruturação e o versionamento do projeto. Estrutura Básica Em seguida, preparei a estrutura básica do projeto. Sem utilizar JavaScript, foquei em criar diretórios para os templates e recursos estáticos, como CSS e imagens, que seriam fundamentais para a interface do usuário. Criação de Postagens Para armazenar as informações das postagens, como título, descrição e autor, criei um arquivo JSON. Esta abordagem facilitou a manipulação e o armazenamento de dados de forma estruturada. Geração de Páginas Com o uso do Flask, desenvolvi um sistema para ler os dados dos arquivos markdown e gerar páginas HTML estáticas para cada postagem. Este processo foi central para transformar o conteúdo em formato markdown em páginas web visualmente atraentes. Integração do Gitflow Implementei o Gitflow, criando branches específicas para desenvolvimento, features e releases. Essa metodologia me ajudou a manter o projeto organizado e facilitou o trabalho colaborativo. Personalização e Estilo Personalizei o estilo do blog com CSS, focando em criar um design que fosse ao mesmo tempo simples e elegante, proporcionando uma experiência de usuário agradável. Teste Local Realizei testes locais para verificar se as postagens eram geradas corretamente e se o estilo visual estava de acordo com o planejado. Esses testes foram cruciais para assegurar a qualidade do produto final. Publicação no GitHub Pages Após finalizar o desenvolvimento e os testes, fiz o push do projeto para o GitHub e habilitei o GitHub Pages, publicando assim o blog na internet. Documentação do Procedimento Como parte do projeto, documentei detalhadamente como cada etapa foi concebida e realizada. Esta documentação foi publicada como o primeiro post do blog, servindo como um guia para quem deseja seguir um caminho semelhante.
